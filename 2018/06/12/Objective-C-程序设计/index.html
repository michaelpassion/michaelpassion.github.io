Objective-C 程序设计

发表于 2018-06-12

讲一个变量定义为特定类的对象时，使用的是静态类型。 “静态”指的是对存储在变量中对象的类型进行显示声明。使用静态类型时，编译器尽可能确保变量的用法在程序中始终保持一致。编译器能够通过检查来确定应用于对象的方法是有该类定义的还是有该类集成的，否则它将显示警告信息。

关心静态类型是因为它能更好地在程序编译阶段而不是在运行时指出错误。

特殊类型 instancetype 表明，从init方法返回的类型与它的初始化类（也就是初始化消息的接收者） 相同。这位编译器提供了辅助，它可以检查你的程序和标记的潜在类型是否匹配。

关于属性、存取方法和实例变量

目前的趋势是使用下划线作为实例变量名字起始字符。通过X错的生成的模板代码可以看到，引用实例变量的变量名字是以 ‘_’开头的

遇到 @synthesize 指令如 @synthesize window = window; 表明合成属性 window 的取值方法和设值方法，并将属性与实例变量 _window (实例变量并没有显示声明)关联起来。这对区别属性和实例变量的使用时有帮助的，鼓励通过设值方法来设值实例变量的值，通过取值方法获取实例变量。
